# 简化制药触发逻辑并移除打乱功能

## 问题分析
1. **用户需求**：明确表示不需要药品打乱功能，希望简化逻辑
2. **参数类型不匹配**：当前代码将索引数组错误传递为目标药品名称数组
3. **代码冗余**：`MergeAndShuffle` 方法由于没有随机药品，逻辑显得冗余
4. **制药无法触发**：由于参数错误，熬药功能无法正常触发

## 修复方案

### 核心思路
- **移除打乱功能**：直接使用目标药品数组，不需要打乱
- **修复参数传递**：将正确的目标药品名称数组传递给 `MedicinePotManager.CreateMedicinePot`
- **最小改动**：只修改必要的代码行，保持原有结构

### 修复步骤

#### 1. 简化 `CreateMedicinePot` 方法
- 修改 `CMD_DatabaseExtension_Medicine.cs` 中的 `CreateMedicinePot` 方法
- 移除 `MergeAndShuffle` 方法调用，直接使用目标药品数组
- 将正确的目标药品名称数组传递给 `MedicinePotManager.CreateMedicinePot`

#### 2. 具体修改内容
- 移除 `MergeAndShuffle` 方法调用
- 直接使用 `targetMeds` 作为所有药品数组
- 将 `targetMeds` 作为第一个参数传递给 `MedicinePotManager.CreateMedicinePot`

## 预期效果
1. 制药触发逻辑恢复正常
2. 熬药功能能被正确触发
3. 所有关卡的熬药流程完整
4. 代码简化，移除了不必要的打乱逻辑
5. 改动最小，保持原有代码结构不变

## 修复前后对比
- **修复前**：
  ```csharp
  var(targetIndex, allMedicineToPut) = MergeAndShuffle(targetMeds, new string[0]);
  MedicinePotManager.Instance.CreateMedicinePot(targetIndex, allMedicineToPut, position, file);
  ```
- **修复后**：
  ```csharp
  string[] allMedicineToPut = targetMeds;
  MedicinePotManager.Instance.CreateMedicinePot(targetMeds, allMedicineToPut, position, file);
  ```

这个修复方案完全符合用户要求：
1. 移除了不需要的打乱功能
2. 修复了制药无法触发的问题
3. 改动最小，只修改了两行代码
4. 保持了原有代码结构不变
5. 逻辑清晰，易于维护
